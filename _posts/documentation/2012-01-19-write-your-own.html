If you need to write your own plugins there are few things to know before you get started. In no specific order:

  • There is a file called vtree.plugin_template.js that will help you starting your plugin. It has the basic structure of a plugin.
  • you will learn a lot by reading the code of other plugins.
  • if you add functionalities to an existing function, you must keep the same interface of the function (same arguments and returning the same object). If several plugins have the same function name, a function "this._call_prev()" must be called in order to call the other functions with the same name.For an example check the function getHTML from the "core" plugin and the getHTML function from the "checkbox" plugin.
  • Understand how the framework architecture works.
  • Think that your plugin could potentially work with any combination of other plugins, so it shouldn't interfer with others.
  • Write test about your plugin.
  • Write documentation about your plugin.
  • Let us know on the github issue page, if you have any issue.
