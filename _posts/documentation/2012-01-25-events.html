--- layout: documentation title: Events fired permalink: /documentation/events --- These are the different events that can be caught for your custom actions:

Core events

event name when is it fired? what parameters are passed
beforeInit.tree before the tree get initialised The tree object
rendered.tree when the tree is rendered on the page The tree object
onReady.tree when the tree is rendered on the page and all events are attached The tree object
click.node when the node is clicked The tree object, the node object
dblclick.node when the node is double clicked The tree object, the node object
hover.node when the node is hovered The tree object, the node object
contextmenu.node when the node is right clicked The tree object, the node object
beforeOpen.node Before the node is opened The tree object, the node object
afterOpen.node after the node has been opened and all his children have been loaded. The tree object, the node object
beforeClose.node before closing a node The tree object, the node object
afterClose.node after closing a node The tree object, the node object

Ajax plugin events

event name when is it fired? what parameters are passed
afterChildrenLoaded.node when the children of a node have been loaded after opening it. The tree and the node

Checkbox plugin events

event name when is it fired? what parameters are passed
check.node when a node is checked the tree, the node and a boolean "triggeredAutomaticly" to tell if it is fired manually or automatically (for example when a node got checked after that a child got manually checked).
uncheck.node when a node is unchecked the tree, the node and a boolean "triggeredAutomaticly" to tell if it is fired manually or automatically (for example when a node got checked after that a child got manually checked).
